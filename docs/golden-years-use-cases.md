## 参与者（Actor）总览

| Actor | 类型 | 说明 |
|-------|------|------|
| 社工 | Primary | 一线上门服务人员，执行探访关爱、助浴、助医陪诊等服务 |
| 站长 | Primary | 站点管理者，负责排班调度、服务监控、日志审核 |
| 总部管理员 | Primary | 跨站点管理，全局配置与数据分析 |
| 家属 | Primary | 老人的家庭成员，接收健康报告、查看趋势、留言 |
| 服务主管/SOP负责人 | Primary | 负责SOP设计、培训组织、凭证标准制定 |
| 需求受理人员 | Primary | 负责接听电话、记录外生需求、初步分诊 |
| 质检/审核/结算人员 | Primary | 独立于站长的审核角色，负责凭证合规和结算对接 |
| 系统 | Secondary | 实时督导、异常检测、日志/报告自动生成、智能提醒等后台能力 |
| 微信/短信平台 | Secondary | 消息推送渠道 |
| 保险公司/政府/机构结算方 | Secondary | 接收导出的凭证包，完成结算（对接方） |

## 核心业务对象

| 业务对象 | 说明 |
|----------|------|
| 站点 | 服务站点，归属机构，包含地址、区域、服务覆盖范围 |
| 社工（含技能/班次/培训状态） | 一线服务人员，记录技能资质、排班班次、培训完成情况、可用状态 |
| 老人（含服务资格/禁忌） | 服务对象，记录服务资格（养护险/政府购买/机构自费）、频次限制、禁忌事项 |
| 家属 | 老人的家庭联系人，记录授权关系、信息接收渠道偏好 |
| 服务项目 | 可提供的服务类型（探访关爱、助浴、助医陪诊等） |
| SOP | 每种服务项目对应的标准操作流程模板，含步骤、话术、时长、检查项 |
| 项目规则 | 不同资金来源（养护险/政府购买/机构）下的结算标准、审核要求、频次限制 |
| 服务需求 | 待服务的需求条目，来源分内生（定频、异常跟进、健康风险）和外生（电话、小程序、家属、社区转介） |
| 待排班池 | 经分诊确认后等待排班的服务需求集合 |
| 上门任务 | 已排班并分配给社工的具体服务任务 |
| 服务过程 | 社工执行上门服务的完整会话，含到达、SOP执行、异常记录、完成归档 |
| 服务凭证 | 单次服务产出的结构化记录，含对话摘要、SOP完成情况、照片/录音/签字等证据 |
| 服务凭证包 | 按项目规则打包的凭证集合，用于提交给保险公司/政府/机构进行结算 |
| 家属报告 | 面向家属的脱敏健康报告，含健康指标、服务摘要、关注提醒、趋势对比 |
| 反馈/投诉/回访 | 家属或老人的反馈意见、投诉记录，以及主动回访的执行与结果 |

## 主业务链路

```
机构/站点初始化
  → 人员/老人/家属/服务项目准备
  → 培训与知识支持
  → 服务需求进入
  → 分诊确认 → 待排班池
  → 排班派单
  → SOP驱动的上门服务
  → 服务凭证采集 → 服务凭证包生成
  → 服务凭证审核
  → 导出给保险公司/政府/机构结算
  → 家属报告 / 投诉回访 / 老人洞察 / 运营复盘
```

## A. 机构与站点初始化

### UC-B-001 初始化机构与站点

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-B-001 |
| **用例名称** | 初始化机构与站点 |
| **用例描述** | 总部管理员创建机构信息并初始化下属服务站点，配置站点的基础属性、服务覆盖区域和管理人员 |
| **参与者** | 主要：总部管理员<br>次要：— |
| **业务目标** | 建立机构-站点组织架构，为后续人员、老人、服务项目的挂载提供基础 |
| **前置条件** | 总部管理员已登录管理端；机构已完成工商注册/资质审批 |
| **后置条件** | 机构信息已创建；至少一个站点已初始化并可投入使用 |
| **触发条件** | 新机构上线；新增服务站点；站点信息变更 |
| **基本流程** | 1. 总部管理员进入机构管理页面<br>2. 录入机构基础信息：名称、统一社会信用代码、资质类型、联系人<br>3. 创建下属站点：站点名称、地址、所属行政区域、服务覆盖范围（街道/社区）<br>4. 为站点指定站长<br>5. 配置站点的服务能力：可提供的服务项目列表、最大社工人数<br>6. 系统校验信息完整性并保存<br>7. 站点状态置为"已就绪"，可开始录入社工和老人信息 |
| **备选流程** | 3a. 批量导入多个站点信息 → 系统解析导入文件并逐条校验<br>4a. 站长尚未录入系统 → 先跳转到社工管理（UC-B-002）创建站长账号后再回来指定 |
| **异常流程** | E1. 站点服务覆盖区域与已有站点重叠 → 系统提醒并要求确认或调整 |
| **业务规则** | 每个站点必须有且仅有一名站长；站点服务覆盖区域不宜重叠；机构信息创建后部分字段（如信用代码）不可修改 |
| **非功能需求** | — |
| **优先级** | High |
| **版本 & 日期** | v0.2 / 2026-05-12 |

## B. 人员与社工管理

### UC-B-002 管理社工档案与可用性

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-B-002 |
| **用例名称** | 管理社工档案与可用性 |
| **用例描述** | 站长或总部管理员维护社工的完整档案信息，包括技能资质、排班班次、培训状态、请假与临时不可用，并支持替补社工的快速指定 |
| **参与者** | 主要：站长、总部管理员<br>次要：— |
| **业务目标** | 确保社工信息完整、可用性实时准确，为排班派单提供可靠的人力资源数据 |
| **前置条件** | 站点已初始化（UC-B-001） |
| **后置条件** | 社工档案已创建或更新；可用性状态已同步到排班模块 |
| **触发条件** | 新社工入职；社工信息变更；社工请假/临时不可用；培训完成/资质到期 |
| **基本流程** | 1. 管理员进入社工管理页面<br>2. 录入/编辑社工基础信息：姓名、手机号、身份证号、所属站点<br>3. 配置社工技能与资质：持有技能标签（探访关爱、助浴、助医陪诊等）、资质证书编号及有效期<br>4. 设置排班班次：默认工作时段、每日最大服务户数<br>5. 维护培训状态：已完成培训列表、待培训项、培训到期提醒<br>6. 管理可用性：当前状态（在岗/请假/培训中/临时不可用）、请假时段、不可用原因<br>7. 设置替补关系：为社工指定默认替补人选（技能匹配、区域就近）<br>8. 系统保存并同步可用性到排班模块 |
| **备选流程** | 6a. 社工临时请假 → 系统自动检查其名下待执行任务，提示站长是否启用替补<br>7a. 未指定替补 → 系统根据技能和区域自动推荐替补候选 |
| **异常流程** | E1. 删除仍有活跃排班的社工 → 系统阻止并提示先处理排班<br>E2. 社工资质到期 → 系统标记为"资质过期"，排班时自动排除并提醒站长 |
| **业务规则** | 社工必须至少持有一项技能标签才能参与排班；资质过期的社工不可分配对应类型的服务任务；总部管理员可管理所有站点社工，站长仅可管理本站点 |
| **非功能需求** | — |
| **优先级** | High |
| **版本 & 日期** | v0.2 / 2026-05-12 |

## C. 老人/家属档案与服务资格

### UC-B-004 管理老人档案与服务资格

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-B-004 |
| **用例名称** | 管理老人档案与服务资格 |
| **用例描述** | 管理员维护老人的完整档案，包括基础信息、健康档案、服务资格（养护险/政府购买/机构自费）、服务频次限制和禁忌事项。服务完成后系统自动追加健康数据 |
| **参与者** | 主要：站长、总部管理员<br>次要：系统（自动更新健康数据） |
| **业务目标** | 维护准确的老人档案和服务资格数据，支撑排班、服务执行、凭证审核和结算 |
| **前置条件** | 站点已初始化（UC-B-001） |
| **后置条件** | 老人档案已创建或更新；服务资格信息可供排班和结算模块使用 |
| **触发条件** | 老人初次签约服务；老人信息变更；服务资格变更；每次服务完成后自动更新健康数据 |
| **基本流程** | 1. 管理员进入老人管理页面<br>2. 录入/编辑基础信息：姓名、性别、年龄、身份证号、地址、联系电话<br>3. 建立健康档案：基础病史、用药情况、身体指标基线、过敏史<br>4. 配置服务资格：资金来源类型（养护险/政府购买/机构自费）、对应的服务项目范围<br>5. 设置频次限制：每种服务类型的周/月最大服务次数（依据资格类型和项目规则）<br>6. 录入禁忌事项：不可提供的服务类型、特殊注意事项（如对某类清洁剂过敏）<br>7. 录入专属注意事项（如独居、有宠物、门锁密码等）<br>8. 系统保存档案 |
| **备选流程** | 8a. 每次服务完成后，系统自动将本次采集的健康数据追加到档案中（如最新血压值、体重），不覆盖人工录入的基线数据<br>4a. 老人同时具备多种服务资格 → 系统支持多资格并存，排班和结算时按规则选择优先资格 |
| **异常流程** | E1. 服务资格到期 → 系统标记为"资格过期"，排班时提醒站长，不自动生成新需求<br>E2. 频次限制已满 → 系统在排班时自动拦截，提示本周期剩余可用次数 |
| **业务规则** | 健康档案是服务执行的核心上下文来源；不同服务资格对应不同的项目规则和结算标准；禁忌事项在排班和服务启动时必须向社工提醒 |
| **非功能需求** | — |
| **优先级** | High |
| **版本 & 日期** | v0.2 / 2026-05-12 |

| **参与者** | 主要：站长、总部管理员<br>次要：家属 |

| **触发条件** | 老人签约时录入家属信息；家属信息变更；家属授权调整 |

| **异常流程** | E1. 家属联系方式无效（微信未关注公众号/手机号错误） → 系统标记并提醒管理员核实 |
| **业务规则** | 每位老人至少关联一位家属作为主要联系人；健康报告仅推送给有查看授权的家属；紧急情况通知所有已关联家属 |
| **非功能需求** | — |
| **优先级** | High |
| **版本 & 日期** | v0.2 / 2026-05-12 |

## D. 服务类型、SOP与项目规则

### UC-B-006 管理服务项目与SOP模板

| 字段 | 内容 |
|------|------|
| **用例编号** | UC-B-006 |
| **用例名称** | 管理服务项目与SOP模板 |
| **用例描述** | 服务主管创建、编辑、启用或停用服务项目及其对应的SOP模板，定义每种服务类型的标准流程、引导话术和凭证要求（照片/录音/签字/时长等） |
| **参与者** | 主要：服务主管/SOP负责人、总部管理员<br>次要：— |
| **业务目标** | 标准化各类上门服务的流程和质量要求，明确每种服务的凭证采集标准和完成判定标准 |
| **前置条件** | 机构已初始化（UC-B-001） |
| **后置条件** | 服务项目和SOP模板已创建/更新，可用于排班、服务执行和凭证审核 |
| **触发条件** | 新增服务类型；现有SOP需要优化调整；凭证标准变更 |
| **基本流程** | 1. 服务主管进入服务项目管理页面<br>2. 创建或编辑服务项目：名称、类别、适用人群、预计时长范围<br>3. 编辑SOP模板：步骤列表（每步包含：步骤名、引导话术、检查项、预期时长）<br>4. 配置凭证要求：该服务类型需要采集的证据类型（拍照/录音/老人签字/服务时长记录），每种证据的数量和质量要求<br>5. 设定完成判定标准：必须完成的SOP步骤列表、最低服务时长、必须采集的凭证项<br>6. 服务主管保存模板<br>7. 模板生效，可在排班和服务中使用 |
| **备选流程** | 2a. 复制已有模板 → 在副本上修改，快速创建变体<br>5a. 按服务资格类型设置差异化完成标准（如养护险要求签字+照片，机构自费仅需时长记录） |
| **异常流程** | E1. 停用正在排班中引用的模板 → 系统提醒有关联排班，确认后停用，已排班任务不受影响<br>E2. 删除已有服务记录引用的服务项目 → 系统阻止删除，仅允许停用 |
| **业务规则** | 每个服务项目必须关联一个有效的SOP模板；凭证要求是审核和结算的依据；已停用的模板不影响历史记录 |
| **非功能需求** | — |

  └──→ 需求进入待排班池 → UC-B-012 制定排班计划

  └──→ 下发上门任务到 UC-B-014

UC-B-013 调整排班任务
  └──→ 更新 UC-B-014 的任务队列

UC-B-014 查看今日服务任务
  └──→ UC-B-015 启动上门服务

UC-B-015 启动上门服务
  ├──→ UC-B-016 执行上门服务
  └── 加载上下文：老人档案(UC-B-004) + SOP模板(UC-B-006) + 家属留言(UC-B-026) + 禁忌事项

UC-B-016 执行上门服务
  ├── <<extend>> UC-B-017 记录服务异常
  └──→ UC-B-018 完成服务并归档

UC-B-018 完成服务并归档
  ├── <<include>> UC-B-019 生成服务日志与凭证
  └── <<include>> UC-B-024 生成家属健康报告

UC-B-019 生成服务日志与凭证
  └──→ 供 UC-B-020 审核 / UC-B-021 导出

UC-B-020 审核服务凭证
  └──→ 通过后供 UC-B-021 导出凭证包

UC-B-021 导出服务凭证包
  └──→ 提交给保险公司/政府/机构结算

UC-B-022 处理投诉与质疑
  ├──→ 可生成新服务需求 → UC-B-010
  └──→ 可触发SOP优化(UC-B-006) / 培训(UC-B-008)

UC-B-023 发起并执行主动回访
  ├──→ 可生成新服务需求 → UC-B-010
  └──→ 可生成投诉工单 → UC-B-022

UC-B-024 生成家属健康报告
  └──→ 推送到 UC-B-025 家属查看

UC-B-027 查询服务统计与运营复盘
  ├──→ 反馈到 UC-B-008 培训改进
  ├──→ 反馈到 UC-B-006 SOP优化
  └──→ 反馈到 UC-B-012 排班策略调整
