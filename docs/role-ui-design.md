# 分角色 UI 设计

状态：Role UI design baseline
日期：2026-05-13
仓库：`aro-network/lumii-fieldwork`

## 1. UI 框架

每个角色进入后都是：

```text
聊天主界面
  + 当前角色相关 Tab
```

聊天用于提问、查询、总结、解释、提醒和生成建议。

Tab 用于展示结构化数据、记录明细、报告、待确认事项和可执行动作。

红配参考只取这个组织方式：主界面是聊天，结构化内容放进规整 Tab，不使用到处漂浮的 panel。

## 2. 集团管理

默认聊天问题：

- 最近服务质量怎么样？
- 社工人员有没有按规程做？
- 哪些站点异常最多？
- 哪些服务对象需要重点关注？

Tabs：

| Tab | 内容 |
| --- | --- |
| 运营总览 | 服务量、完成率、异常率、投诉率、凭证通过率 |
| 规程执行 | SOP 完成率、漏项排行、社工人员对比、站点对比 |
| 服务质量 | 服务中断、时长异常、异常未闭环、质检问题 |
| 服务对象洞察 | 状态变化、风险趋势、重点关注对象 |
| 报告 | 周报、月报、专题分析、导出 |

主要动作：

- 追问数据原因。
- 下钻到站点、社工人员、服务对象和服务记录。
- 生成整改建议。
- 导出报告。

## 3. 站点运营

默认聊天问题：

- 今天有哪些服务？
- 哪些工牌还没归属？
- 哪些服务记录需要确认？
- 哪些异常还没处理？

Tabs：

| Tab | 内容 |
| --- | --- |
| 工牌设备 | 工牌状态、绑定机构、可用性、丢失状态 |
| 服务任务 | 排单、待服务、已完成、未匹配任务 |
| 服务记录 | 录音生成的服务记录、归属置信度、待补充字段 |
| 待确认事项 | 低置信度归属、异常、服务中断、SOP 漏项 |
| 异常跟进 | 异常记录、处理状态、回访和整改 |

主要动作：

- 激活工牌。
- 可选分配社工人员。
- 可选排单。
- 确认低置信度归属。
- 跟进异常和整改。

## 4. 服务主管

默认聊天问题：

- 这个服务项目 SOP 怎么设置？
- 最近哪些服务没有按规程做？
- 哪些社工人员需要复训？
- 哪些异常说明 SOP 需要调整？

Tabs：

| Tab | 内容 |
| --- | --- |
| SOP | 服务项目、步骤、必问项、异常分支、完成标准 |
| 规程执行 | SOP 检查结果、漏项记录、服务记录样本 |
| 服务质量 | 服务完整性、语音中断、异常闭环、质检反馈 |
| 培训与整改 | 复训对象、整改建议、SOP 更新记录 |

主要动作：

- 设置或更新 SOP。
- 查看规程执行情况。
- 复盘异常服务。
- 生成培训和整改建议。

## 5. 社工人员

默认聊天问题：

- 今天我要服务谁？
- 当前服务有什么注意事项？
- 我刚才的记录完整吗？

Tabs：

| Tab | 内容 |
| --- | --- |
| 今日任务 | 可选排单产生的任务、地址、时间、注意事项 |
| 当前服务 | 工牌记录状态、开始时间、服务对象、SOP 提醒 |
| 服务对象信息 | 基本信息、禁忌、上次异常、家属关注点 |
| 历史记录 | 自己的服务记录、待补充事项、异常说明 |

主要动作：

- 查看今日任务。
- 查看当前服务提示。
- 补充服务说明。
- 查看自己的服务记录。

说明：

- 最小使用方式不要求社工人员打开界面。
- 社工人员只要按工牌按钮开始和结束，后台也能生成服务记录。

## 6. 质检人员

默认聊天问题：

- 哪些记录需要审核？
- 哪些记录证据不完整？
- 哪些服务疑似没有按规程做？

Tabs：

| Tab | 内容 |
| --- | --- |
| 待审核记录 | 待审服务记录、凭证、录音和转写 |
| 异常记录 | 时长异常、语音中断、服务对象归属异常 |
| 规程漏项 | SOP 漏项、服务质量问题、整改建议 |
| 凭证导出 | 凭证包、导出记录、结算交接 |

主要动作：

- 审核服务记录。
- 退回补充。
- 标记问题。
- 导出凭证包。

## 7. 服务对象家属

默认聊天问题：

- 最近服务情况怎么样？
- 家里老人状态有没有变化？
- 我想补充一个需求。

Tabs：

| Tab | 内容 |
| --- | --- |
| 服务报告 | 脱敏服务摘要、服务时间、服务内容 |
| 状态变化 | 身体、情绪、生活状态和风险提示 |
| 反馈与需求 | 家属反馈、投诉、补充需求、回访记录 |

主要动作：

- 查看服务报告。
- 查看状态变化。
- 提交反馈或新需求。
- 请求回访。
