# 分角色 UI 设计

状态：Role UI design baseline
日期：2026-05-13
仓库：`aro-network/lumii-fieldwork`

## 1. UI 框架

本文只对齐 [`agentic-flows.md`](agentic-flows.md)。每个角色进入后都是：

```text
聊天主界面
  + 当前角色相关 Tab
```

聊天用于提问、查询、总结、解释、提醒和生成建议。

Tab 用于展示结构化数据、记录明细、报告、待确认事项、订阅状态和可执行动作。

红配参考只取这个组织方式：主界面是聊天，结构化内容放进规整 Tab，不使用到处漂浮的 panel，不要求用户先理解后台模块。

后台 Agent 能力不作为独立用户角色呈现。F05 自动规整和 F06 周期报告只通过各角色的记录、报告、洞察、待确认和订阅报告入口暴露。

## 2. Flow 对照

| Flow | 主角色 | UI 承接 |
| --- | --- | --- |
| F01 运营激活智能工牌 | 站点运营 | 站点运营：工牌设备 |
| F02 服务主管设置 SOP | 服务主管 | 服务主管：SOP |
| F03 运营排单 | 站点运营 | 站点运营：服务任务 |
| F04 社工人员上门记录服务 | 社工人员 | 社工人员：当前服务、历史记录 |

| F06 后台周期性生成洞察报告 | 后台 Agent | 集团管理：报告；服务主管：SOP 调整；家属：订阅报告 |
| F07 家属订阅、实时查询和接收报告 | 服务对象家属 | 服务对象家属：服务与状态、订阅报告、反馈与需求 |

## 3. 集团管理

默认聊天问题：

- 最近服务质量怎么样？
- 社工人员有没有按规程做？
- 哪些站点异常最多？
- 哪些服务对象需要重点关注？
- 家属订阅和反馈趋势怎么样？

Tabs：

| Tab | 内容 |
| --- | --- |
| 运营总览 | 服务量、完成率、异常率、投诉率、结算日志导出趋势 |
| 质量与规程 | SOP 完成率、漏项排行、服务中断、时长异常、异常未闭环、站点对比 |
| 对象与家属 | 服务对象状态变化、风险趋势、重点关注对象、家属订阅、反馈、投诉、回访请求 |
| 报告 | 周报、月报、专题分析、导出管理报告 |

主要动作：

- 追问数据原因。
- 下钻到站点、社工人员、服务对象和服务记录。
- 查看服务对象风险、家属订阅和反馈趋势。
- 生成整改建议。
- 导出管理报告。

边界：

- 集团管理看跨站点趋势和管理报告，不作为 F08 结算服务日志的默认导出角色。

## 4. 站点运营

默认聊天问题：

- 今天有哪些服务？
- 哪些工牌还没归属？
- 哪些服务记录需要确认？
- 哪些异常还没处理？
- 本月哪些服务日志可以导出结算？
- 哪些家属反馈需要处理？

Tabs：

| Tab | 内容 |
| --- | --- |
| 工牌设备 | 工牌状态、绑定机构、可用性、丢失状态、社工人员可选分配 |
| 服务任务 | 排单、待服务、已完成、未匹配任务 |
| 服务记录 | 录音生成的服务记录、归属置信度、待补充字段、异常标记、结算字段筛选、批量导出、导出历史 |
| 待确认事项 | 低置信度归属、异常、服务中断、SOP 漏项、缺失字段 |
| 家属反馈 | 家属追问、补充需求、投诉、回访请求、处理状态 |

主要动作：

- 激活工牌。
- 可选分配社工人员。
- 可选排单。
- 确认低置信度归属和缺失字段。
- 跟进异常、整改和家属反馈。
- 按需导出服务日志用于结算。

服务记录内导出规则：

- 导出前必须展示筛选条件、字段范围、异常标记和未确认项。
- 导出后记录操作者、导出时间、筛选条件和文件版本。
- 存在低置信度归属、未闭环异常或缺失字段时，必须提示先确认或带标记导出。

## 5. 服务主管

默认聊天问题：

- 这个服务项目 SOP 怎么设置？
- 哪些异常分支需要补进 SOP？
- 哪些 SOP 版本需要调整？

Tabs：

| Tab | 内容 |
| --- | --- |
| SOP | 服务项目、步骤、必问项、异常分支、完成标准、版本和生效范围 |
| SOP 调整 | 后台汇总的漏项模式、异常分支触发、服务记录样本引用、调整建议、版本变更记录 |

主要动作：

- 设置或更新 SOP。
- 查看需要调整 SOP 的汇总证据。
- 调整 SOP 版本和生效范围。
- 发布 SOP 更新。

边界：

- 服务主管负责 SOP 定义、版本和调整，不审核单条服务记录，不退回记录，也不负责 F08 服务日志结算导出。

## 6. 社工人员

默认聊天问题：

- 今天我要服务谁？
- 当前服务有什么注意事项？
- 我刚才的记录完整吗？

Tabs：

| Tab | 内容 |
| --- | --- |
| 今日任务 | 可选排单产生的任务、地址、时间、注意事项 |
| 当前服务 | 工牌记录状态、开始时间、服务对象、对象注意事项、禁忌、上次异常、家属关注点、SOP 提醒、实时反馈 |
| 历史记录 | 自己的服务记录、待补充事项、异常说明 |

主要动作：

- 查看今日任务。
- 查看当前服务提示和服务对象上下文。
- 按工牌开始和结束服务记录。
- 补充服务说明。
- 查看自己的服务记录。

说明：

- 最小使用方式不要求社工人员打开界面。
- 社工人员只要按工牌按钮开始和结束，后台也能生成服务记录。
- 没有排单时，社工人员仍可直接服务，后台按录音内容推断服务对象和服务项目。

## 7. 服务对象家属

入口：

- 小程序。
- 公众号。

默认聊天问题：

- 最近服务情况怎么样？
- 今天有没有上门服务？
- 家里老人状态有没有变化？
- 这周/月服务报告发我看一下。
- 我想补充一个需求。

Tabs：

| Tab | 内容 |
| --- | --- |
| 服务与状态 | 最近服务状态、服务时间、服务摘要、状态变化、风险提示、后续关注事项 |
| 订阅报告 | 订阅状态、日报、周报、月报、推送记录 |
| 反馈与需求 | 家属反馈、投诉、补充需求、回访请求、处理进度 |

主要动作：

- 绑定服务对象并订阅小程序或公众号消息。
- 实时查询服务情况。
- 查看日报、周报和月报。
- 追问报告内容。
- 提交反馈、投诉或新需求。
- 请求回访。

隐私边界：

- 家属只看脱敏服务摘要、服务时间、服务内容、状态变化和风险提示。
- 家属不能看到原始录音、原始转写、内部审核备注、社工人员绩效、后台置信度和机构内部处理记录。
