# 分角色 UI 设计

状态：Role UI design baseline
日期：2026-05-13
仓库：`aro-network/lumii-fieldwork`

## 1. UI 框架

本文只对齐 [`agentic-flows.md`](agentic-flows.md)。每个角色进入后都是：

```text
聊天主界面
  + 当前角色相关 Tab
```

聊天用于提问、查询、总结、解释、提醒和生成建议。

Tab 用于展示结构化数据、记录明细、报告、待确认事项、订阅状态和可执行动作。

红配参考只取这个组织方式：主界面是聊天，结构化内容放进规整 Tab，不使用到处漂浮的 panel，不要求用户先理解后台模块。

后台 Agent 能力不作为独立用户角色呈现。F05 自动规整和 F06 周期报告只通过各角色的记录、报告、洞察、待确认和订阅报告入口暴露。

## 2. Flow 对照

| Flow | 主角色 | UI 承接 |
| --- | --- | --- |
| F01 运营激活智能工牌 | 站点运营 | 站点运营：工牌设备 |
| F02 服务主管设置 SOP | 服务主管 | 服务主管：SOP |
| F03 运营排单 | 站点运营 | 站点运营：服务任务 |
| F04 社工人员上门记录服务 | 社工人员 | 社工人员：当前服务、历史记录 |
| F05 后台自动规整服务记录 | 后台 Agent | 站点运营：服务记录、待确认事项；质检人员：记录审核 |
| F06 后台周期性生成洞察报告 | 后台 Agent | 集团管理：运营报告；服务主管：执行复盘；家属：订阅报告 |
| F07 家属订阅、实时查询和接收报告 | 服务对象家属 | 服务对象家属：实时查询、订阅报告、反馈与需求 |
| F08 运营导出服务日志用于结算 | 站点运营 | 站点运营：服务记录中的筛选和导出动作 |
| F09 运营和管理查看报告并主动查询 | 集团管理、站点运营、服务主管、质检人员 | 各角色聊天入口和结构化 Tab |

## 3. 集团管理

默认聊天问题：

- 最近服务质量怎么样？
- 社工人员有没有按规程做？
- 哪些站点异常最多？
- 哪些服务对象需要重点关注？
- 家属订阅和反馈趋势怎么样？

Tabs：

| Tab | 内容 |
| --- | --- |
| 运营总览 | 服务量、完成率、异常率、投诉率、结算日志导出趋势 |
| 规程执行 | SOP 完成率、漏项排行、社工人员对比、站点对比 |
| 服务质量 | 服务中断、时长异常、异常未闭环、质检问题 |
| 服务对象洞察 | 状态变化、风险趋势、重点关注对象 |
| 家属反馈 | 订阅量、日报/周报/月报触达、反馈、投诉、回访请求 |
| 运营报告 | 周报、月报、专题分析、导出管理报告 |

主要动作：

- 追问数据原因。
- 下钻到站点、社工人员、服务对象和服务记录。
- 查看家属订阅和反馈趋势。
- 生成整改建议。
- 导出管理报告。

边界：

- 集团管理看跨站点趋势和管理报告，不作为 F08 结算服务日志的默认导出角色。

## 4. 站点运营

默认聊天问题：

- 今天有哪些服务？
- 哪些工牌还没归属？
- 哪些服务记录需要确认？
- 哪些异常还没处理？
- 本月哪些服务日志可以导出结算？
- 哪些家属反馈需要处理？

Tabs：

| Tab | 内容 |
| --- | --- |
| 工牌设备 | 工牌状态、绑定机构、可用性、丢失状态、社工人员可选分配 |
| 服务任务 | 排单、待服务、已完成、未匹配任务 |
| 服务记录 | 录音生成的服务记录、归属置信度、待补充字段、异常标记、结算字段筛选、批量导出、导出历史 |
| 待确认事项 | 低置信度归属、异常、服务中断、SOP 漏项、缺失字段 |
| 家属反馈 | 家属追问、补充需求、投诉、回访请求、处理状态 |

主要动作：

- 激活工牌。
- 可选分配社工人员。
- 可选排单。
- 确认低置信度归属和缺失字段。
- 跟进异常、整改和家属反馈。
- 按需导出服务日志用于结算。

服务记录内导出规则：

- 导出前必须展示筛选条件、字段范围、异常标记和未确认项。
- 导出后记录操作者、导出时间、筛选条件和文件版本。
- 存在低置信度归属、未闭环异常或缺失字段时，必须提示先确认或带标记导出。

## 5. 服务主管

默认聊天问题：

- 这个服务项目 SOP 怎么设置？
- 最近哪些服务没有按规程做？
- 哪些异常说明 SOP 需要调整？

Tabs：

| Tab | 内容 |
| --- | --- |
| SOP | 服务项目、步骤、必问项、异常分支、完成标准、版本和生效范围 |
| 执行复盘 | SOP 检查结果、漏项记录、服务完整性、语音中断、异常闭环、服务记录样本、SOP 调整证据 |

主要动作：

- 设置或更新 SOP。
- 查看 SOP 执行复盘。
- 复盘异常服务。
- 生成 SOP 调整建议。

边界：

- 服务主管围绕 SOP 定义和执行复盘工作，不承接 agentic flow 之外的独立人员管理需求，也不负责 F08 服务日志结算导出。

## 6. 社工人员

默认聊天问题：

- 今天我要服务谁？
- 当前服务有什么注意事项？
- 我刚才的记录完整吗？

Tabs：

| Tab | 内容 |
| --- | --- |
| 今日任务 | 可选排单产生的任务、地址、时间、注意事项 |
| 当前服务 | 工牌记录状态、开始时间、服务对象、对象注意事项、禁忌、上次异常、家属关注点、SOP 提醒、实时反馈 |
| 历史记录 | 自己的服务记录、待补充事项、异常说明 |

主要动作：

- 查看今日任务。
- 查看当前服务提示和服务对象上下文。
- 按工牌开始和结束服务记录。
- 补充服务说明。
- 查看自己的服务记录。

说明：

- 最小使用方式不要求社工人员打开界面。
- 社工人员只要按工牌按钮开始和结束，后台也能生成服务记录。
- 没有排单时，社工人员仍可直接服务，后台按录音内容推断服务对象和服务项目。

## 7. 质检人员

默认聊天问题：

- 哪些记录需要审核？
- 哪些记录证据不完整？
- 哪些服务疑似没有按规程做？
- 哪些异常影响服务质量判断？

Tabs：

| Tab | 内容 |
| --- | --- |
| 记录审核 | 待审服务记录、服务时间、录音、转写、归属置信度 |
| 异常记录 | 时长异常、语音中断、服务对象归属异常、未闭环事项 |
| 规程漏项 | SOP 漏项、服务质量问题、整改建议 |
| 质检报告 | 审核统计、问题趋势、站点和社工人员对比 |

主要动作：

- 审核服务记录。
- 退回补充。
- 标记问题。
- 追问服务质量和规程执行证据。

边界：

- 质检人员查看记录和问题证据，不作为 F08 结算服务日志的默认导出角色。

## 8. 服务对象家属

入口：

- 小程序。
- 公众号。

默认聊天问题：

- 最近服务情况怎么样？
- 今天有没有上门服务？
- 家里老人状态有没有变化？
- 这周/月服务报告发我看一下。
- 我想补充一个需求。

Tabs：

| Tab | 内容 |
| --- | --- |
| 实时查询 | 最近服务状态、服务时间、服务摘要、后续关注事项 |
| 订阅报告 | 订阅状态、日报、周报、月报、推送记录 |
| 状态变化 | 身体、情绪、生活状态和风险提示 |
| 反馈与需求 | 家属反馈、投诉、补充需求、回访请求、处理进度 |

主要动作：

- 绑定服务对象并订阅小程序或公众号消息。
- 实时查询服务情况。
- 查看日报、周报和月报。
- 追问报告内容。
- 提交反馈、投诉或新需求。
- 请求回访。

隐私边界：

- 家属只看脱敏服务摘要、服务时间、服务内容、状态变化和风险提示。
- 家属不能看到原始录音、原始转写、内部质检备注、社工人员绩效、后台置信度和机构内部处理记录。
