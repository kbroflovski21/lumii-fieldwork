# 护理人员页面产品规格

状态：v2 — 增加工牌联动与当前服务流程
日期：2026-05-17
视觉规范：参照 [global-ui-guidance.md](./global-ui-guidance.md)
业务流程参照：[agentic-flows.md](./agentic-flows.md) F04 社工人员上门记录服务

## 1. 页面定位

护理人员（护工/上门服务人员）的移动工作台。以 H5 页面形式运行，可嵌入 App 或第三方平台（飞书、微信等）。手机端优先设计。

目标用户：一线护理服务人员（护工），需要查看每日任务排期、了解任务详情、学习服务 SOP、配合智能工牌完成上门服务记录。

## 2. 页面结构

```
登录页 → 主页面
              ├── Header（用户信息 + 工牌状态 + 退出登录）
              ├── 当前服务 Banner（条件显示，见第 10 节）
              ├── Tab 1: 我的任务（默认）── iOS 风格日历视图
              │                              └── 点击事件 → 任务详情 Drawer
              │                                                └── 查看报告（已完成任务）
              │                                                └── 地址导航（地图 App 选择）
              ├── Tab 2: 参考资料 ── SOP 列表
              │                       └── 点击条目 → SOP 详情 Drawer
              └── AI 助手（悬浮按钮，常驻所有页面）
```

## 3. 登录页

### 3.1 功能

- 用户名 + 密码登录表单。
- Demo 模式：默认填入用户名"王建国-余杭站-护理服务人员"、密码自动填充（显示为 `••••••`），用户点击"登录"即可进入。

### 3.2 UI

- 居中卡片布局，顶部产品标识（"金色年华"）。
- 输入框高度 44px，遵循 `--radius-sm` 圆角。
- 登录按钮使用 Primary Button 样式（`--accent` 底色）。
- 底部显示"智慧养老服务平台"弱文字。

## 4. 主页面框架

### 4.1 顶部 Header

- 高度 56px。
- 左侧：用户 avatar（首字母彩色底 + 姓氏）+ 姓名 + 角色标签。
- 右侧：工牌状态 chip（见第 10.1 节）+ 退出登录按钮。
- 退出登录：点击弹出二次确认弹窗，确认后回到登录页。

### 4.2 底部 Tab Bar

- 固定底部，两个 Tab：
  - **我的任务**（日历图标）── 默认激活。
  - **参考资料**（书本图标）。
- 图标 + 9-11px 短标签，点击目标高度不小于 44px。
- 激活态：`--accent` 色图标 + 文字；未激活态：`--text-muted`。
- 底部适配 safe-area-inset-bottom。

## 5. 我的任务（日历视图）

### 5.1 导航栏

- 左侧：`← 标题 →` + 快捷按钮（本月/本周/今天）。
- 右侧：Segmented Control `日 | 周 | 月`。
- 默认"周"视图。

### 5.2 月视图

- 顶部：`← 2026年5月 → 本月`。
- 日历网格：7 列（周日~周六），显示日期数字。
- 有任务的日期下方显示小圆点（颜色对应任务状态）。
- 点击某日 → 下方展开该日任务列表。
- 今天日期使用 `--accent` 圆形底色高亮。

### 5.3 周视图

- 顶部：`← 5/17 - 5/23 → 本周`。
- 7 天横向排列，显示 周几 + 日期，今天高亮。
- 下方按天分组显示任务事件列表。

### 5.4 日视图

- 顶部：`← 5月17日 周日 → 今天`。
- 时间轴布局（8:00 - 18:00），每小时一格。
- 任务以色块显示在对应时间段，高度按时长比例。

### 5.5 任务事件颜色

| 状态 | 背景色 | 色条/圆点 |
| --- | --- | --- |
| 已完成 | `rgba(16,185,129,0.1)` 淡绿 | `#10B981` |
| 异常 | `rgba(239,68,68,0.1)` 淡红 | `#EF4444` |
| 待完成（未来） | `rgba(245,158,11,0.1)` 淡橙 | `#F59E0B` |

### 5.6 任务事件摘要格式

```
{服务类型} {服务对象姓名} {时间段} {地点简称}
```

## 6. 任务详情 Drawer

### 6.1 触发方式

点击日历中的任务事件，从底部弹出 Drawer。

### 6.2 Drawer 结构

#### 顶部 Header（带状态色背景条）

- 状态 badge + 服务类型标题 + 日期时间 + 来源标签。
- 底部对应状态色渐变条。右上角关闭按钮。

#### 摘要卡片区

| 卡片 | 内容 |
| --- | --- |
| 服务对象 | Avatar + 姓名 |
| 服务人员 | Avatar + 姓名 |
| 服务地点 | 定位图标 + 完整地址 + **导航按钮** |

- 导航按钮点击弹出地图 ActionSheet（百度/高德/腾讯地图选择）。

#### 排期详情（网格）

服务日期、时间段、服务项目、来源、状态、服务人员、服务对象、地址（含导航按钮）、备注。

#### 服务报告（已完成任务）

- 底部显示"查看报告"入口。
- 点击进入报告子视图（同一 Drawer 内替换）：服务摘要、SOP 执行情况（✓/✗）、健康观察、情绪状态、关注事项、满意度。
- 左上角"← 返回详情"回到任务详情。

### 6.3 Drawer 样式

- 最大高度 86vh，内容区内部滚动。
- `--radius-lg` 上圆角，`--shadow-drawer-mobile` 阴影。
- Scrim overlay 点击关闭。

## 7. 参考资料 Tab

### 7.1 SOP 列表

- 按类型分组：**通用规范**、**服务项目规范**。
- 每个 SOP 条目是 Rich Row：图标 + 标题 + 版本/更新日期 + chevron。
- 点击 → 打开 SOP 详情 Drawer。

### 7.2 SOP 详情 Drawer

- 底部 Drawer，最大高度 86vh。
- Header：SOP 标题 + 关闭按钮。
- 内容区：SOP 全文，Markdown 渲染，支持滚动。

## 8. AI 助手

- 悬浮按钮（FAB）：紫色渐变圆形，固定右下角（Tab 栏上方），z-index 最高层，登录后任何页面状态下可见。
- 点击 FAB 打开全屏聊天面板：紫色渐变 header，向下箭头收起。
- 底部预设快捷问题 + 输入框 + 发送按钮。
- 当前为 mock 回复，后续接入真实 AI。

## 10. 工牌联动与当前服务

### 10.1 工牌状态指示器

Header 右侧常驻显示工牌连接状态：

| 状态 | 显示 | 样式 |
| --- | --- | --- |
| 已连接 · 空闲 | `🟢 工牌在线` | 绿底 chip |
| 已连接 · 录音中 | `🔴 录音中 MM:SS` | 红底 chip + 脉冲圆点 + 计时 |
| 未连接 | `⚪ 工牌离线` | 灰底 chip |
| 未绑定 | 不显示 | — |

工牌通过 BroadcastChannel 与页面通信（原型阶段由硬件模拟器页面模拟）。

### 10.2 当前服务状态机

一条任务的服务流转：

```
空闲 → 服务前提示(T-15min) → 进行中(工牌开始) → 待补充(工牌结束, 30min窗口) → 已完成
```

#### 状态 A：服务前提示（排期开始前 15 分钟触发）

显示在日历视图顶部，橙色 Banner：

- 即将开始 + 开始时间。
- 服务类型 · 服务对象 · 地点。
- 服务对象上下文：饮食禁忌、上次服务情况、家属关注点。
- SOP 要点摘要。

仅有排期的任务触发此提示。临时上门（无排期）不显示提示，工牌直接开始。

#### 状态 B：服务进行中（工牌按钮开始后触发）

显示在日历视图顶部，红色 Banner：

- 服务进行中 + 脉冲圆点 + 计时器。
- 服务类型 · 服务对象 · 地点。
- SOP 进度清单（逐步标记完成，模拟后台实时检查）。
- 服务对象注意事项提醒。

#### 状态 C：待补充（工牌按钮结束后，30 分钟窗口内）

显示在日历视图顶部，绿色 Banner：

- 服务已完成 ✓。
- 服务类型 · 服务对象 · 地点。
- 补充说明输入框（30 分钟内可编辑）。
- 提交按钮。

30 分钟后 Banner 消失，任务转为普通已完成状态。

### 10.3 硬件模拟器

独立页面 `careworkerhardware.html`，用于模拟工牌物理按钮：

- 深色界面，居中大按钮。
- 按钮切换：待命 → 录音中 → 待命。
- 通过 BroadcastChannel 向护理人员页面发送状态事件。
- 在另一个浏览器标签页打开，模拟工牌蓝牙连接。

## 11. 技术实现

- 技术栈：React 18 + TypeScript + Vite + Tailwind CSS。
- 代码位置：`pages/careworker/`。
- 入口页面：
  - `careworker.html` — 护理人员主页面。
  - `careworkerhardware.html` — 工牌硬件模拟器。
- 纯前端原型，无后端依赖，所有数据使用 mock。
- 手机端优先（375px 基准），适配至桌面端。
- 视口适配：`100dvh` + `window.innerHeight` JS fallback + `viewport-fit=cover`。
- 页面间通信：BroadcastChannel（同源跨标签页）。

## 12. 存疑事项

以下事项为产品讨论中提出但尚未确定是否纳入的功能，记录备查。

### 12.1 当前服务流程（已实现为原型，待确认）

**背景**：原始 spec 只包含任务排期查看和 SOP 学习。当前服务流程（服务前提示、进行中面板、补充录入）是基于 agentic-flows F04 补充的，目的是让页面覆盖社工上门服务的完整 end-to-end 工作流。

**待确认点**：

- 服务前提示的触发时机（当前设定为 15 分钟）是否合理？不同服务类型是否需要不同提前量？
- 补充录入的 30 分钟窗口是否足够？是否需要支持超时后的申请延期？
- SOP 进度在"进行中"面板的实时标记，真实场景中由后台 AI 根据录音内容自动检查。原型中为定时模拟，实际需要对接后台 Agent 能力。
- 临时上门（无排期）的服务如何在页面上呈现？当前设计是工牌直接开始，页面仅显示"服务进行中"banner，不显示服务前提示。

### 12.2 工牌状态指示器（已实现为原型，待确认）

**背景**：员工与工牌存在绑定关系。如果员工有绑定的工牌，页面应显示连接与工作状态。

**待确认点**：

- 实际产品中工牌通过蓝牙/Wi-Fi 连接手机，原型使用 BroadcastChannel 模拟。正式实现需要定义设备通信协议。
- 工牌电量、信号强度等硬件信息是否需要在页面展示？
- 工牌未绑定员工时的交互流程（当前设计为不显示 chip）。

### 12.3 历史记录独立视图（决定不做）

**背景**：agentic-flows 角色定义中社工有"历史记录" Tab（自己的服务记录列表、待补充事项、异常说明）。

**决定原因**：当前日历视图已可通过翻页查看历史任务，重复建设价值有限。如果后续任务量增大、需要筛选/搜索/导出历史记录，可以再考虑。

### 12.4 通知与提醒中心（暂不做）

**背景**：后台可自动生成提醒（异常待处理、SOP 漏项、排班变更、家属反馈）。

**暂不做原因**：通知提醒需要在站点管理层面先定义什么行为触发通知、通知的优先级和渠道。这是一个跨角色的系统级设计，不属于护理人员页面单独可以定义的范畴。待站点运营 flow 完善后再考虑。

### 12.5 服务对象详细档案（未涉及）

**背景**：社工在服务前可能需要查看服务对象的完整档案（健康史、用药、过敏、家属联系方式等），而不仅是上下文提示。

**考虑**：当前服务前提示中包含关键注意事项，但完整档案需要后台数据支撑。可作为 AI 助手的查询能力（"陈阿姨的健康档案"）而非独立页面。
