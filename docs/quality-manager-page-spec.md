## 1. 页面定位

集团层面的服务质量监测与分析工作台。用户是集团总部负责质量管控的人员或管理层。

核心动机：
- 监测各站点的服务质量是否达标
- 了解服务 SOP 是否被有效执行
- 掌握客户满意度和异常趋势
- 发现站点之间的质量差异
- 推动形成质量提升的闭环

## 3. 质量总览

### KPI 卡片

6 个指标卡片，响应式网格布局：

| 指标 | 说明 | 趋势颜色逻辑 |
|------|------|-------------|
| 本周服务总量 | 所有站点合计 | 上升=绿 |
| 服务完成率 | 完成/计划 | 上升=绿 |
| SOP 平均完成率 | 所有服务的平均 SOP 执行率 | 上升=绿 |
| 客户满意度 | 5 分制 | 上升=绿 |
| 异常率 | 异常服务/总服务 | 下降=绿（逆向指标） |
| 投诉率 | 投诉数/总服务 | 下降=绿（逆向指标） |

每个卡片：值（22px 加粗）+ 趋势（带颜色）+ 说明文字

### 站点对比表

表格形式，列：站点名、服务量、完成率、SOP 完成率、满意度、异常率。

SOP 和异常率使用语义色标注：
- ≥90% / ≤5%：绿色
- ≥80% / ≤8%：黄色
- <80% / >8%：红色

### 服务项目 SOP 完成率

按服务类型分卡片（如探访关爱、助浴、用药提醒、助餐），每个卡片包含：
- 服务类型名 + SOP 完成率数字
- 进度条（颜色同上）
- 服务次数 + 主要问题说明

## 4. 服务记录

全站点服务记录表格，布局参考站点运营的服务记录页面。

### 工具栏

- 搜索框：搜索社工或服务对象姓名
- 站点筛选下拉：全部站点 / 翠苑站 / 三墩站 / 古荡站 / 文新站
- 状态筛选下拉：全部状态 / 正常 / 警告 / 异常

筛选激活时文字变蓝色加粗（区别于默认灰色）。

### 表格列

| 列 | 说明 |
|----|------|
| 时间 | MM-DD HH:MM 格式 |
| 站点 | 站点名称（此列为质量管理特有，站点运营页面无此列） |
| 社工 | 姓名 |
| 服务对象 | 姓名 |
| 服务项目 | 类型名称 |
| 时长 | XX 分钟 |
| SOP | 完成率百分比，语义色 |
| 满意度 | 文字（满意/非常满意/—） |
| 状态 | 状态标签（正常/警告/异常），语义色 6px 圆角 |

行 hover 时背景变浅灰。

### 表格视觉规范

- 表格容器：白底 + 1px 边框 + 12px 圆角
- 表头：`--surface-subtle` 底色 + 12px 大写加粗字体 + sticky
- 数据行：60px 高度 + `--line-subtle` 分割线
- 状态标签：非 pill 形（6px 圆角），使用语义色背景+文字

## 5. AI 质量助手

### 交互

- 右下角 52px 紫色悬浮圆形图标
- 点击打开 420px 右侧 drawer（带半透明遮罩）
- Drawer 内为完整聊天界面

### AI 能力

由 Qwen3-max 驱动，直接从浏览器调用 DashScope API（静态部署无后端）。

系统 prompt 注入当前所有站点和服务的质量数据，支持：
- 分析某站点质量趋势（「文新站异常率为什么高」）
- 对比不同站点（「对比翠苑站和古荡站的表现」）
- 查看某服务 SOP 执行情况（「助浴的 SOP 主要问题在哪」）
- 给出改进建议

### Drawer 设计

- Header：紫色渐变头像 + 「AI 质量助手」标题 + 关闭按钮
- 消息区：滚动消息列表，AI 消息左对齐+头像，用户消息右对齐蓝色
- 输入区：pill 输入框 + 圆形发送按钮
- 「正在思考...」加载状态

## 6. 与其他页面的关系

| 关联页面 | 数据流 |
|---------|--------|
| 服务主管 | SOP 定义和督导策略 → 质量管理查看 SOP 执行率 |
| 社工上门服务 | 服务记录 + SOP 完成率 + 异常标记 → 汇总到质量报表 |
| 站点运营 | 站点级服务数据 → 质量管理做跨站点对比 |
| 家属 | 客户满意度反馈 → 汇总到质量 KPI |
