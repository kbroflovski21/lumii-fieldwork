# 家属页面 — 产品设计规格

状态：v1 设计定稿
日期：2026-05-18
仓库：`aro-network/lumii-fieldwork`
视觉规范：`docs/global-ui-guidance.md`
部署：`https://kbroflovski21.github.io/lumii-fieldwork/pages/family/`

## 1. 页面定位

家属通过微信服务号/小程序访问的移动端 H5 页面。家属是服务对象（老人）的亲属，核心诉求是了解老人的服务情况、接收服务报告、反馈意见。

页面设计面向微信生态：类似微信服务号推送消息的体验，无需复杂操作。

## 2. 目标用户

服务对象的家属（如张大伟的儿子张明）。

使用场景：
- 收到微信推送后打开查看服务报告
- 主动查看老人最近的服务情况
- 向服务站点反馈意见或提出需求

## 3. 页面结构

移动端单列布局，底部固定两个 Tab：

| Tab | 功能 |
|-----|------|
| 📢 最新动态 | 微信服务号风格的推送消息流 |
| 💬 反馈建议 | 智能客服对话式反馈 |

### Header

固定顶部，包含：
- 服务对象头像/图标 + 姓名 + 「的服务」
- 家属身份（如「张明（儿子）」）+ 站点名

## 4. 最新动态 Tab

按消息推送形式展示，类似微信服务号的消息列表。每条消息是一个可展开的卡片。

### 消息类型

| 类型 | 标签颜色 | 内容 |
|------|---------|------|
| 服务报告 | 蓝色（accent） | 某次上门服务完成后的 AI 摘要（社工、时间、时长、健康数据、SOP 完成率） |
| 健康周报 | 绿色（success） | 周期性健康状态总结（血压趋势、情绪、运动能力、综合建议） |
| 通知 | 黄色（warning） | 排班调整、服务时间变更等运营通知 |
| 月度汇总 | 紫色（aiAccent） | 月度服务量、健康趋势、综合评估 |

### 卡片交互

- 收起状态：标签 + 日期 + 标题 + 摘要预览（60 字）
- 展开状态：完整内容（Markdown 文本，支持换行）
- 点击切换收起/展开

### 数据来源

生产环境中，消息由后台系统在以下时机自动生成并推送：
- 社工完成一次服务 → 生成服务报告
- 每周日 → 生成健康周报
- 每月 1 日 → 生成月度汇总
- 运营手动发布 → 通知

## 5. 反馈建议 Tab

智能客服对话风格，非结构化表单。

### 开场白

客服小张开场：
> 您好，我是客服小张 👋
> 有什么反馈或建议都可以告诉我，比如：
> · 对服务内容或时间的建议
> · 对服务人员的评价
> · 希望增加或调整的服务
> · 任何疑问或投诉
> 我们保证翠苑站的运营主管会看到并处理您的每一条留言。

### 交互

- 家属输入文字 → 发送 → 客服回复确认收到（「收到您的反馈...这条留言会转给运营主管处理」）
- 纯对话交互，无表单、无按钮选项
- 每条反馈在后台转给站点运营的家属反馈列表

### 技术实现

Demo 中为 mock 回复。生产环境可接入 LLM 智能客服或人工客服系统。

## 6. 视觉规范

- 最大宽度 480px，居中对齐（适配手机和 PC 预览）
- 底部 Tab 栏高度 52px，固定定位
- 颜色 token 遵循 `global-ui-guidance.md`
- 消息卡片：白底 + 1px 边框 + 12px 圆角
- 客服头像：32px 圆形，浅蓝底 + 👩‍💼 emoji
- 用户消息：蓝色背景 + 白字 + 右对齐
- 客服消息：白底 + 灰边框 + 左对齐

## 7. 与其他页面的关系

| 关联页面 | 数据流 |
|---------|--------|
| 社工上门服务 | 社工完成服务 → 生成服务报告 → 推送到家属「最新动态」 |
| 站点运营 | 家属提交反馈 → 流转到站点运营「家属反馈」列表 |
| 集团质量管理 | 家属满意度数据汇总到集团质量报表 |
