## 3. 关键 Flow 明细

### F01 外生电话需求受理

触发：老人、家属或社区工作人员来电。

责任线：管理。

涉及传统角色：需求受理人员、站长。

自治等级：Copilot。

Agent 做什么：

- 把接线员输入的自然语言转成结构化需求。
- 提示缺失信息，例如老人、地址、联系人、服务类型、期望时间。
- 查询老人服务资格、历史风险和近期服务。
- 建议紧急程度和分诊方向。

人工确认点：

- 创建外生需求。
- 将需求放入待排班池。
- 标记不可服务或关闭。
- 启动紧急升级路径。

输出：

- 企业聊天通道中可快速记录和追问。
- web 需求面板中形成待分诊需求。

### F02 内生服务需求生成

触发：固定频次到期、周期计划、异常复访、健康风险、长期未服务。

责任线：管理。

涉及传统角色：站长、调度。

自治等级：Full autonomous。

Agent 做什么：

- 定时扫描老人服务频次、上次服务、异常复访规则和风险标签。
- 主动生成待确认内生需求。
- 解释需求来源和推荐服务时间。

人工确认点：

- 进入待排班池。
- 关闭或延期。

输出：

- 企业聊天通道提醒站长“明日有 18 个到期服务需求”。
- web 需求池展示来源、原因和优先级。

### F04 排班草案生成与派单确认

触发：待排班池有未安排需求，或站长授权生成排班草案。

责任线：管理。

涉及传统角色：站长、调度、社工管理。

自治等级：Delegated。

Agent 做什么：

- 读取待排班需求、社工可用性、服务技能、路线、时长、老人风险。
- 生成排班草案、路线顺序、冲突说明和替代方案。
- 发现请假、培训不满足、任务超时和路线不合理。

人工确认点：

- 派单。
- 改派。
- 取消。
- 下发到社工 H5。

输出：

- web 排班面板展示草案、冲突和确认动作。
- 企业聊天通道发送提醒和打开排班面板链接。

### F05 社工服务前 hint

触发：社工打开今日任务、准备出发或到达前。

责任线：督导。

涉及传统角色：一线社工、服务主管。

自治等级：Full autonomous。

Agent 做什么：

- 主动汇总老人注意事项、上次异常、家属留言、服务项目凭证要求。
- 给出本次服务最关键的 3-5 条 hint。
- 提醒需要带的物料、设备、电量、耳机和录音权限。

人工确认点：

- 社工确认已读关键风险。
- 特殊服务项目可能要求确认物料已准备。

输出：

- 社工 H5 任务详情页。
- 必要时企业聊天通道提醒。

### F06 服务中 SOP 漏项提醒

触发：服务进行中，转写或社工操作显示 SOP 关键项未完成。

责任线：督导。

涉及传统角色：社工、服务主管。

自治等级：Copilot。

Agent 做什么：

- 跟踪 SOP 状态。
- 识别漏问、漏拍、漏记和超时风险。
- 通过耳机或 H5 给出短提醒。

人工确认点：

- 社工是否采纳提醒。
- 必做项是否完成。

输出：

- 耳机短提示。
- 社工 H5 当前提醒和 SOP 进度。

### F07 现场异常分支升级

触发：老人不在家、拒绝服务、身体异常、情绪异常、环境风险、需要通知家属或拨打 120。

责任线：督导 / 管理。

涉及传统角色：社工、站长、服务主管、家属沟通人员。

自治等级：Copilot / Confirm-required。

Agent 做什么：

- 按 SOP 异常分支提示社工下一步。
- 判断是否需要通知站长、家属或外部紧急机构。
- 自动形成异常记录草稿。

人工确认点：

- 异常类型。
- 是否已处理。
- 是否通知站长或家属。
- 是否取消、改约或升级。

输出：

- 社工 H5 异常分支。
- 企业聊天通道提醒站长。
- web 异常面板留痕。

### F09 服务凭证包生成与缺证据提醒

触发：服务完成。

责任线：合规能力。

涉及传统角色：质检、审核、结算、站长。

自治等级：Delegated。

Agent 做什么：

- 汇总服务对象、服务人员、服务项目、时间线、SOP 完成、照片、录音/转写引用、异常处理、审核记录。
- 根据项目规则生成服务凭证包草案。
- 主动发现缺照片、缺备注、时长不足、异常未处理和规则不满足。

人工确认点：

- 退回补充。
- 标记结算就绪。
- 导出凭证包。

输出：

- web 凭证面板。
- 导出文件。
- 企业聊天通道发送缺证据提醒和打开链接。

### F11 家属报告生成与发送

触发：服务完成并满足报告发送条件。

责任线：洞察。

涉及传统角色：家属沟通、站长、服务主管。

自治等级：Delegated / Confirm-required。

Agent 做什么：

- 基于服务事实和老人历史生成脱敏家属报告草案。
- 转成家属能读懂的服务摘要、状态变化和关注提醒。
- 排除内部服务日志、原始敏感转写、社工绩效和审核备注。

人工确认点：

- 首版 MVP 默认发送前人工确认。
- 后续可按服务类型配置自动发送规则。

输出：

- web 报告预览。
- 家属 H5 报告链接。
- 企业聊天通道只发送提醒和链接。

### F12 投诉与主动回访

触发：家属反馈、投诉、异常服务后、长期未反馈、高风险老人。

责任线：洞察 / 管理。

涉及传统角色：家属沟通、回访人员、站长、服务主管。

自治等级：Copilot / Delegated。

Agent 做什么：

- 将家属留言分类为普通反馈、投诉、新需求或注意事项。
- 生成回访问题清单。
- 生成回访任务草案。
- 将回访结果转为老人档案补充或新服务需求草案。

人工确认点：

- 关闭投诉。
- 创建新服务需求。
- 向家属发送正式处理结论。

输出：

- 企业聊天通道用于接收和提醒。
- web 回访/投诉面板处理闭环。

### F14 培训与查手册

触发：社工主动提问、新 SOP 发布、服务前需要注意事项、服务中不确定。

责任线：督导。

涉及传统角色：服务主管、社工、站长。

自治等级：Copilot / Full autonomous。

Agent 做什么：

- 回答社工自然语言问题，并引用 SOP、手册或项目规则。
- 新 SOP 或新项目规则发布后提醒相关社工学习。
- 服务前主动推送本次任务相关手册片段。

人工确认点：

- 培训合格状态。
- SOP 或规则发布。

输出：

- 企业聊天通道问答。
- 社工 H5 服务前和服务中快速查询。
- web 培训/知识面板。

## 4. UI mock 对应关系

当前 UI mock 覆盖以下视角：

- 企业聊天通道：响应和主动提醒入口，不绑定飞书卡片。
- 管理工作台：需求池、排班、异常、凭证、回访。
- 督导工作台：SOP、培训、服务前 hint、服务中提醒。
- 洞察工作台：家属报告、老人趋势、投诉回访。
- 社工 H5：任务、hint、服务中提醒、异常、完成。
- 家属 H5：脱敏报告和反馈入口。
